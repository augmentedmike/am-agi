# Step 4: Agent Visibility

The agent is powered by Claude Code running in print mode — a non-interactive loop that takes a prompt, does work, and exits. All input and output is structured JSON. No human in the loop per iteration.

## How Claude Code Runs

```sh
claude --dangerously-skip-permissions -p "$(cat prompt.md)" --output-format json > iter/<n>/output.json
```

Key flags:
- `--dangerously-skip-permissions` — runs without interactive permission prompts, required for automation
- `-p` — print mode; takes a prompt, runs once, exits
- `--output-format json` — structured output, pipeable

Each iteration is one invocation. The runner assembles the prompt (system prompt for the current column + current file state), fires Claude Code, and captures the output.

## The Work Log

Every iteration produces a log file written by the agent: `iter/<n>/agent.log`. This is the agent's account of what it did — decisions made, files changed, commands run, failures encountered. It is committed to git alongside all other artifacts from that iteration.

The log is not optional. It is the primary record of what happened inside the agent's execution and the basis for human review and debugging.

## Git as Document Safety

Git is the persistence layer for everything the agent produces. After every iteration, the runner commits all changes:

```sh
git add -A
git commit -m "<task-slug>/iter-<n>: <one-line summary>"
```

Every commit includes:
- All code and file changes from the iteration
- `iter/<n>/agent.log` — the agent's work log
- Updated `todo.md`
- Any generated or modified documents (`criteria.md`, `research.md`, etc.)

Nothing is lost. Every state the agent passed through is recoverable from git history. If an iteration produces bad output, you roll back. If you want to understand why the agent made a decision three iterations ago, you read that commit.

## The Ship Script

When a card reaches `shipped`, a deterministic script takes over. No model involvement — pure shell:

```sh
# Squash all iteration commits into one clean commit
git reset $(git merge-base HEAD origin/main)
git add -A
git commit -m "<task-slug>: <description>"

# Rebase onto current main and merge
git fetch origin
git rebase origin/main
git checkout main
git merge --ff-only <task-slug>
git push origin main

# Clean up
git worktree remove ../am-<task-slug>
git branch -d <task-slug>
```

This script is always the same. It does not depend on model output or judgment. The result is a single atomic commit on main representing the complete task — all iteration history collapsed inside it, clean linear trunk preserved.

## What You Can See

| Artifact | Where |
|---|---|
| What the agent did each iteration | `iter/<n>/agent.log` |
| What the agent plans to do next | `todo.md` |
| What the agent decided "done" means | `criteria.md` |
| What sources the agent used | `research.md` |
| Full execution history | `git log` on the task branch |
| Final clean record | Single commit on `main` |

The agent has no hidden state. Everything it knows, decided, or produced is in the filesystem and in git.

## Prompt

```
Read steps/4.md, docs/AGENT-LOOP.MD, and CLAUDE.md. Build two things:

**1. Claude Code invocation wrapper — `agent/src/claude/invoke.ts`**

A typed async function `invokeClaudeCode(prompt: string, iterDir: string): Promise<InvokeResult>` that:
- Runs: `claude --dangerously-skip-permissions -p <prompt> --output-format json`
- Streams stdout to `<iterDir>/output.json`
- Parses the JSON output and extracts the agent's log text
- Writes the log text to `<iterDir>/agent.log`
- Returns `{ success: boolean, log: string, outputPath: string }`
- On non-zero exit: returns success=false with stderr captured in log, does not throw

**2. Git commit helper — `agent/src/git/commit.ts`**

Two exported functions:

`commitIteration(cardId: string, iterN: number, summary: string): Promise<void>`
- Runs: `git add -A && git commit -m "<cardId>/iter-<n>: <summary>"`
- Extracts the one-line summary from the agent log if not provided

`shipCard(cardId: string, description: string): Promise<void>`
- Squashes all iteration commits: `git reset $(git merge-base HEAD origin/main)`
- Commits: `git commit -m "<cardId>: <description>"`
- Rebases: `git fetch origin && git rebase origin/main`
- Merges: `git checkout main && git merge --ff-only <cardId>`
- Pushes: `git push origin main`
- Cleans up: `git worktree remove ../am-<cardId> && git branch -d <cardId>`
- Each step is its own named function — if any step fails it throws with the step name and stderr

Code quality:
- All shell commands go through a single `exec(cmd: string): Promise<ExecResult>` utility — never raw Bun.spawn scattered through the code
- ExecResult type: `{ stdout: string, stderr: string, exitCode: number }`
- exec throws only on unexpected errors (spawn failure); non-zero exit codes are returned, not thrown
- No hardcoded paths — workDir and repoRoot are parameters
- Commit messages are validated before use: must be non-empty, under 72 chars for the subject line

Definition of done:
- `bun test` passes for: output JSON parsing, log extraction, commit message formatting, shipCard step sequencing (mock exec)
- invokeClaudeCode called against a real Claude Code install writes output.json and agent.log to the iter dir
- shipCard called on a test branch with 3 iteration commits produces exactly one commit on main and removes the worktree and branch

To execute this step, create a work directory, write this prompt to `work.md`, then run:
`am <workDir>`
```
