# Step 9: The First Real Card

Every step so far has built the system. This step uses it. Step 9 is the first real card — not a test seed, not scaffolding, but actual product work tracked through the board from creation to ship.

The work is the board itself. Meta: the board improves itself by becoming the first card it ships.

## What a Real Card Looks Like

A real card is created with a title and a work description, not a test ID. It goes through every gate. Research gets done. Criteria get written. Tests get written. The agent does one unit of work per iteration and commits. The gate worker verifies before anything ships.

This is the system working end-to-end for the first time on real work.

## The Active Indicator

When an agent is working on a card, the card should show it. A pulsing dot is a live signal — it tells the human watching the board that something is happening right now. This is mechanistic interpretability at the UI level: the internal state of the system (agent running vs idle) is visible on the surface.

For this step, cards in the `in-progress` state with an active agent show the indicator. The simplest correct implementation: any card currently `in-progress` shows the pulse — the runner will be the source of truth for "active" in a later step.

## The Slide Panel

Cards on the board are summaries. The full picture — work log, attachments, criteria, iteration history — lives in a detail panel. Sliding it in from the right keeps context without navigating away. The panel renders card content as Markdown so work logs and criteria read cleanly.

## Implementation Notes for AM

Read steps/9.md and CLAUDE.md before starting.

**Step 1 — Create the card**

```sh
board create --title "Board card UI: glass design, slide panel, active indicator" --priority high
```

Save the returned ID. This card is the work item. All work in this step happens against it.

**Step 2 — Work the card through the board**

Work it through backlog → in-progress → in-review → shipped using the standard agent loop. The work description is the prompt above. The card's worktree is where the implementation lives.

**Implementation requirements (derived from the prompt):**

Card tile (`apps/board/src/components/CardTile.tsx`):
- Show title in large, clean typography — the title is the primary affordance
- Show priority badge using the existing glass treatment
- Show card ID in small muted text below the title
- Pulsing dot: show a `relative flex h-3 w-3` ping animation (Tailwind `animate-ping`) when `card.workDir` is set — indicates an agent is actively attached to the card, regardless of state
- Click anywhere on the card to open the detail panel
- Glass aesthetic: `bg-zinc-800/60 backdrop-blur-sm border border-white/10 rounded-xl` with hover lift

Detail panel (`apps/board/src/components/CardPanel.tsx` — new file):
- Slides in from the right: fixed overlay, panel anchored to right edge, `translate-x-full` → `translate-x-0` transition
- Width: `w-full max-w-xl`
- Close on backdrop click or ESC key
- Renders full card content as Markdown: title, state, priority, ID, work log entries, attachments list, timestamps
- Use a Markdown renderer — `react-markdown` is already available or install it
- Glass panel background: `bg-zinc-900/95 backdrop-blur-md border-l border-white/10`

Top bar:
- Show total count of `in-progress` cards: "N active"
- Pulsing dot next to the count if N > 0

Board layout (`apps/board/src/components/BoardClient.tsx`):
- Wire up card click → open panel with selected card
- Pass selected card state down to CardColumn/CardTile

**Definition of done:**
- Card created on the board and shipped through all gates
- Cards show title, priority badge, and ID
- In-progress cards show pulsing dot
- Clicking a card opens the slide panel
- Panel renders card content in Markdown
- Panel closes on backdrop click and ESC
- Top bar shows active card count with pulse when > 0
- `bun run dev` starts clean, no TypeScript errors
- `bun test` passes
