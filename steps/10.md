# Step 10: Dispatcher, Polling, and New Card Panel

Three things that complete the loop:

1. **Dispatcher** — watches the board and runs AM automatically on cards, feeding the right column prompt per state
2. **Board polling** — the UI refetches cards every 5 seconds so state changes are always visible even if SSE drops
3. **New card panel** — a [new card] button on the top bar opens a panel that slides down from the bar, textarea for work description, creates card in backlog on submit

---

## Prompt

```
Read steps/10.md, apps/board/src/worker/prompts.ts, CLAUDE.md, and docs/TOOLS.md.

Do these three things in order. Commit after each.

**1. Build `bin/dispatcher`**

A Bun script (`#!/usr/bin/env bun`) that watches the board and runs AM iterations on active cards.

- Runs a loop every 5 seconds
- Each cycle: GET /api/cards, filter to backlog/in-progress/in-review, pick highest-priority card per state (critical > high > normal > low)
- For each card, check if worktree `../am-<cardId>` exists
- If not: `git worktree add ../am-<cardId> -b <cardId>`
- Write `work.md` to the worktree each cycle:

```
# Card: <title> (<id>)
State: <state>
Priority: <priority>

## Instructions
<column prompt for current state>
```

- Column prompts (import from `apps/board/src/worker/prompts.ts`):
  - backlog → BACKLOG_PROMPT
  - in-progress → IN_PROGRESS_PROMPT
  - in-review → IN_REVIEW_PROMPT
- Call `runIteration(worktreeDir)` from `agent/src/loop/index.ts`
- Log: `[dispatch] <cardId> <state> iter <n>`
- On error: log and continue, never crash
- AM uses `board move <id> <state>` to transition — dispatcher never touches card state

**2. Add polling to the board UI**

In `apps/board/src/components/BoardClient.tsx`, add a `setInterval` that fetches `/api/cards` every 5 seconds and updates the cards state. Keep the existing SSE — polling is the fallback.

**3. Create the new card panel**

`[new card]` button on the top bar right (pink accent: `bg-pink-500 hover:bg-pink-400`). Clicking toggles a panel that slides down from the top bar. Panel has a single textarea (auto-resize: grows with content, no scrollbar, `resize-none overflow-hidden`, onInput sets height to scrollHeight). A `[create]` button POSTs to `/api/cards` with `{ title: textareaValue }`, closes the panel, clears the textarea. New card appears in backlog via SSE/polling.

**4. Create the card**

After building and committing everything, create a card on the board:

```sh
board create --title "New card panel: [new card] button on top bar, slide-down panel, auto-resize textarea" --priority high
```

Definition of done:
- `bin/dispatcher` starts, picks up cards from the board, feeds column prompts
- Board UI refreshes cards every 5 seconds
- [new card] button visible on top bar, pink accent
- Panel slides down, textarea auto-resizes, submit creates card in backlog
- Card created on board for the new card panel work
- `bun test` passes
```
