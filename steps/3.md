# Step 3: The Runner

The runner loops AM until the agent responds "DONE".

```
am <workDir> → runIteration → runIteration → runIteration → DONE
```

---

## Prompt

```
Read docs/AGENT-LOOP.MD, steps/3.md, and CLAUDE.md in this repo. Build two things:

**1. The runner at `agent/src/runner/index.ts`** using Bun and TypeScript.

The runner is the agent loop. It calls `runIteration` from `agent/src/loop/index.ts` directly in a loop until Claude outputs "DONE". maxIterations defaults to 20.

Export a `runLoop(workDir: string, maxIterations?: number): Promise<void>` function.

Implementation:
1. Call `runIteration({ workDir })`, capture the result
2. Extract the `result` field from the output as the summary — use the first non-empty line only, truncated to 72 chars
3. Commit: `git add -A && git commit -m "iter: <summary>"`
4. If result contains "DONE" → exit 0
5. If `runIteration` throws → print error, exit 1
6. If iteration count reaches maxIterations → print "max iterations reached", exit 1
7. Otherwise loop

**2. `bin/am`** — a thin CLI entry point (`#!/usr/bin/env bun`) that:
- Reads a work description string from `process.argv[2]`, prints usage and exits 1 if missing
- Derives a slug from the description (lowercase, spaces to hyphens, strip non-alphanumeric)
- Creates a git worktree: `git worktree add ../am-<slug> -b <slug>`
- Writes the description to `../am-<slug>/work.md`
- Calls `runLoop(workDir)` from `../agent/src/runner/index.ts`
- Make it executable with `chmod +x bin/am`

Usage: `am "description of the work"`

Code quality:
- Each step is a clearly named function
- No board, no card state, no gate logic
- Tests use real temp directories, no mocks

Definition of done:
- `bun test` passes
- There is an integration test using a real temp git repo that stubs `runIteration` with a fake that returns a result, and asserts a git commit is produced after each iteration
- Runner exits 0 when result contains "DONE"
- Runner exits 1 when `runIteration` throws or max iterations reached

When the implementation is complete and tests pass, commit all source files: `git add -A && git commit -m "step 3: build runner"`
```
