# Step 7: Board Housekeeping and Visual Design

A system that looks good is a system people use. An agent that works inside an ugly, cluttered workspace makes worse decisions — not because the model cares about aesthetics, but because the human overseeing it does. Legibility and visual clarity are part of system quality.

## Cleaning the Board

Test cards are scaffolding — they exist to verify that the system works and should not outlive that purpose. Leaving them on the board creates noise: future agents will encounter them, count them, and potentially act on them. Hygiene is a form of correctness.

The `board archive` command exists precisely for this. Archiving is preferred over deletion — the records remain inspectable in `board/archive/` without polluting the active workspace.

## Visual Design Principles

The board is a control surface, not a dashboard. It needs to communicate state at a glance, sustain hours of use without fatigue, and get out of the way when the agent is working.

### One Accent Color

A single accent color does more semantic work per pixel than a palette. When everything important uses the same color, the eye learns quickly what to pay attention to. Buttons, active tabs, progress indicators, and interactive affordances all share the accent — the user never has to decode a color legend.

Pink is a strong choice here: high contrast against grey without the aggression of red or the urgency-signaling of orange.

### Zinc Grey + Apple Glass

Zinc grey (`zinc-*` in Tailwind) sits in the neutral family — cooler and more refined than slate, less sterile than true grey. Against it, the Apple glass treatment (translucent backgrounds, blur, subtle borders, layered depth) creates hierarchy without hard edges.

The visual grammar: dark zinc base, frosted glass cards and panels floating above it, pink for everything interactive. The result is a surface that reads as a system — not a toy.

### Full-Height Columns

A Kanban board where columns don't span the viewport is not a Kanban board — it's a list with columns painted on. Each column should own its vertical space, scroll independently when content overflows, and remain stable while adjacent columns scroll. This is what the layout enforces.

### Collapsible Shipped Column

Shipped cards are done. They earn their place in the historical record but not in the primary working view. Collapsing the shipped column to a thin vertical strip — "Shipped (n)" rotated 90° — removes it from the working surface while keeping the count visible. One click restores it. The state machine continues to enforce the shipped state; the UI simply gets it out of the way.

**Design System**

Base palette: Tailwind 4 zinc scale (zinc-900 background, zinc-800/zinc-700 for panels and cards).

Glass treatment: card and panel backgrounds use a translucent zinc with `backdrop-blur`. Subtle `border border-white/10` on glass surfaces. Light inner shadow to lift surfaces off the base layer.

Accent: a single pink — use `pink-500` as the base. All interactive elements use this accent:
- Buttons (primary)
- Active/selected tab indicators
- Progress bars
- Focus rings
- Highlighted state badges

Typography: system sans-serif stack. Zinc-100 for primary text, zinc-400 for secondary/muted text.

**Layout**

Board view is a full-viewport flex-row layout:
- `height: 100vh`, `overflow: hidden` on the board container
- Each column: `flex-1`, `h-full`, `overflow-y-auto`, `flex-col`
- Columns are separated by a subtle `border-r border-white/5`
- Column header is sticky at the top: `sticky top-0`

Shipped column is collapsible:
- Default state: collapsed — renders as a thin vertical strip (`w-12`), full height, showing "Shipped (n)" text rotated 90° (`writing-mode: vertical-rl`)
- Expanded state: full column width, same layout as other columns
- A toggle control (chevron or arrow) switches between states
- Collapsed/expanded state persists in `localStorage` under the key `board:shipped-collapsed`
- Card count is always visible in both states

**Card Design**

Cards use the glass treatment: `bg-zinc-800/60 backdrop-blur-sm border border-white/10 rounded-xl`
Hover state lifts the card: `hover:bg-zinc-700/70 hover:border-white/20`
Priority badges:
- critical: `bg-red-500/20 text-red-300 border border-red-500/30`
- high: `bg-orange-500/20 text-orange-300 border border-orange-500/30`
- normal: `bg-zinc-500/20 text-zinc-300 border border-zinc-500/30`
- low: `bg-blue-500/20 text-blue-300 border border-blue-500/30`

**globals.css**

Replace the existing globals.css entirely. Define the zinc/glass design tokens as CSS custom properties inside `@theme`. The base background is zinc-950. No light mode — this is a dark-only control surface.

- Board renders with zinc/glass aesthetic, no default Next.js styles visible
- Board container is exactly viewport height with no outer scrollbar
- Each column scrolls independently when card count exceeds viewport height

- Collapsed state persists across page reloads via localStorage
