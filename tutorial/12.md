# Step 12: Cross-Platform Install

AM ran on macOS from day one. Step 12 makes it run anywhere — Mac, Linux, Windows, or any VPS.

This step adds install scripts for Linux (with init system auto-detection) and Windows (Task Scheduler), a pinned `DEPENDENCIES.md`, a locked Bun version, and moves the board app from `board/` to `board/` to match the `agent/` layout.

---

## What gets built

**Linux install script (`launchagents/linux/install.sh`)**
- Installs Node 24 via nvm, Bun (pinned), Git, Claude CLI
- Detects init system from PID 1: systemd → OpenRC → runit → upstart → fallback
- Registers services with the right mechanism per init system
- Fallback: writes `~/.am/launch.sh` sourced from `~/.profile`

**Supported init systems**

| Init | Distros | Mechanism |
|---|---|---|
| systemd | Ubuntu, Debian, Fedora, Arch, RHEL, VPS | `~/.config/systemd/user/` |
| OpenRC | Alpine, Gentoo, Artix-OpenRC | `~/.am/sv/` run scripts |
| runit | Void Linux, Artix-runit | `~/sv/` run scripts |
| upstart | Legacy Ubuntu < 15.04 | `~/.init/` conf files |
| fallback | Anything else | `~/.am/launch.sh` via `~/.profile` |

**Windows install script (`launchagents/windows/install.ps1`)**
- Installs deps via winget: Node.js LTS, Git; Bun via PS1 installer; Claude CLI via npm
- Registers `AM-Board` and `AM-Dispatcher` as Task Scheduler tasks
- RestartCount 999, 5s interval, no execution time limit
- Logs to `%TEMP%\am-board.log` and `%TEMP%\am-dispatcher.log`

**`DEPENDENCIES.md`**
- Pinned versions for all runtime deps: Node 24.14.0, Bun 1.3.6, Git 2.x, Claude CLI
- npm package table with versions from `board/package.json`
- Per-platform install instructions
- Linux init system support table

**`.bun-version`**
- Pins Bun to `1.3.6` for the whole repo

**`board/` rename**
- Moved from `board/` → `board/` (matching `agent/` layout)
- Updated all path references: install scripts, `agent/src/git/commit.ts`, `scripts/dispatcher.ts`, `bunfig.toml`
- `apps/` directory removed

---

## Lessons Learned

**`board/` was never actually there**

The previous session started from inside `board/`, but git tracking was always under `board/`. The `board/` path was stale Next.js artifacts. Removing `apps/` and updating path references was all that was needed.

**Windows Task Scheduler env vars**

Task Scheduler doesn't inherit shell env. `BOARD_URL` must be injected via a `cmd.exe /c set VAR=val & command` wrapper — you can't use `Environment` directly in `Register-ScheduledTask` without the older XML API.

**Bun version pinning**

`.bun-version` is read by `bun upgrade` and `bun install` to enforce the pinned version. Install scripts should still pass the version explicitly to the install curl command as a fallback for systems without an existing Bun.
