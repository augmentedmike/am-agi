# Step 11: Multi-Project, Settings, Context Budget

The board was AM's own board — one project, one context. Step 11 adds multi-project support, a settings panel, per-card URL sharing, clipboard copy of card IDs, and agent context budget tracking.

This step was built entirely by AM using the dispatcher from step 10.

---

## What gets built

**Multi-project routing**
- `/p/[id]` pages — one URL per project, rendered server-side
- `ProjectSelector` navigates via Next.js router instead of fetching
- `projects` state lifted to `BoardClient` (single source of truth); passed as props to both `ProjectSelector` and `ProjectSettings`

**Settings modal**
- Gear icon always visible in top bar
- When a real project is selected: editable name, `repoDir`, rename syncs workspace directory on disk via `renameSync`
- When `AM Board` is selected: locked read-only panel — all fields display as static text with a "locked" badge

**Card detail URL sync**
- `?card=<id>` appended to current path via `window.history.replaceState` on card open/close
- On mount, `BoardClient` restores selected card from URL param

**Card ID clipboard copy**
- Card panel header shows the truncated card UUID with a copy icon
- `navigator.clipboard.writeText` on click; icon swaps to checkmark for 1.5s

**Agent context budget**
- 40% threshold at 80K tokens (Sonnet 4.6 = 200K window)
- `ClaudeUsage` interface parsed from Claude CLI JSON output
- `appendUsageToIterLog` writes a `## context-usage` section to the current `iter/<n>/agent.log`
- System prompt instructs agent to stop at 40%, commit with `partial — context limit reached`, write progress to `todo.md`, and exit

**Dispatcher safety guard**
- `runIteration` throws if `workDir/.git` is a directory (main repo root, not a worktree)

---

## Lessons Learned

**`createPortal` to escape `backdrop-filter` stacking context**

The settings modal was clipped by the glass-card `backdrop-filter` on parent elements. Fix: render via `createPortal(modal, document.body)` so the modal lives outside the compositing tree.

**Dropdown doesn't update after rename**

`ProjectSelector` had its own `projects` state fetched independently. After a rename, the selector didn't know to refetch. Fix: lift `projects` to `BoardClient`, pass as a prop, use `onProjectUpdated` callback to propagate changes.

**AM Board workspace is the repo root itself**

`workspaces/am-board/` holds non-code artifacts (docs, media, notes) — gitignored. The code lives at repo root. The settings panel shows this path as read-only, not editable.

## Prompt

```
Read steps/11.md, CLAUDE.md, and the current board source.

Do the following. Commit after each.

**1. Multi-project URL routing**

- Create `src/app/p/[id]/page.tsx` — server component that fetches project + cards by id, renders `<BoardClient initialCards={cards} initialProjectId={id} />`
- Add `initialProjectId?: string | null` prop to `BoardClient`; use `useRouter` to navigate on project select
- Lift `projects: Project[]` state to `BoardClient`; pass as props to `ProjectSelector` and `ProjectSettings`

**2. Card URL sync**

- In `BoardClient`, add `useEffect` that calls `window.history.replaceState` when `selectedCard` changes
- On mount, restore `selectedCard` from `?card=` param

**3. Card ID copy**

- In `CardPanel`, add a copy icon next to the truncated card ID in the header
- `navigator.clipboard.writeText(card.id)` on click; swap to checkmark for 1.5s via `copied` state

**4. Settings modal**

- `ProjectSettings` takes `project: Project | null` and `onProjectUpdated` as props
- When `project === null`: render `AmBoardSettingsModal` — locked, read-only, shows repo root path
- When `project` present: editable modal; PATCH `/api/projects/[id]` on save; `renameSync` workspace dir in the API route when `repoDir` changes

**5. Agent context budget**

- Add `ClaudeUsage` to `agent/src/loop/types.ts`
- Parse `usage` from Claude CLI JSON output in `invoke-claude.ts`
- Add `appendUsageToIterLog` to `agent/src/loop/index.ts` — appends `## context-usage` to `iter/<n>/agent.log`
- Add to system prompt: stop at 40% context (≈80K tokens), commit progress, exit DONE
- Add worktree safety guard to `runIteration`: throw if `.git` is a directory

Definition of done:
- `/p/<id>` routes work; AM Board is at `/`
- Renaming a project updates the dropdown immediately and renames the workspace dir
- Card ID copies to clipboard
- Settings modal locked for AM Board, editable for other projects
- Agent logs context usage after each iteration
- `bun test` passes
```
