# Step 1

Init your agent by running `/init` for Claude Code.

---

## Prompt

```
Read init.sh and README.MD in this repo. Your job is to produce a CLAUDE.md that makes every future Claude Code session in this repo immediately productive with zero setup friction.

The CLAUDE.md must cover:
- The one required bootstrap command (`source ./init.sh`) and why it must run first
- What `bin/` is, why it is gitignored, and how it gets populated
- The full agent loop flow at a glance (one-shot iterations, filesystem state, git commits)
- The kanban state machine (four states, gated transitions, priority order)
- Every `board` CLI command with its flags and what it does
- The ship script — exact commands, in order
- The core design principles: CLI governs process, no model memory, computed state over stored state, worktree isolation

Rules:
- No filler, no obvious instructions, no generic dev advice
- Every sentence earns its place — if it can be derived by reading the code, cut it
- Prefer tables over prose for reference material
- Write for an agent starting cold with zero context, not a human reading docs

Definition of done:
- A new Claude Code session sourcing only CLAUDE.md can orient itself and begin productive work without reading any other file
- Every board command is documented with its exact flag syntax
- The ship script is present verbatim
- The file is under 200 lines
```
